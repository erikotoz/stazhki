# Этап 2 — запуск в Телеграме

Бот: **@stazhki_v_raschete_bot**. Приложение откроется прямо в Телеграме, вход — без пароля
(Телеграм сам говорит, кто ты). Снаружи это уже не сайт: по голому адресу будет заглушка.

Что нужно сделать (по порядку). Свой адрес на Render обозначу как
`https://ТВОЁ.onrender.com` — подставь реальный.

---

## 1. Взять токен бота

В Телеграме открой **@BotFather** → `/mybots` → выбери **@stazhki_v_raschete_bot** →
**API Token** → скопируй строку вида `8123456789:AA....`. Это секрет — в чат/код не клади.

## 2. Сбросить базу (один раз)

Схема поменялась (появились группы). В прод-базе лежат только наши тестовые аккаунты,
их не жалко. В Supabase → **SQL Editor** → выполни:

```sql
DROP TABLE IF EXISTS sessions, expenses, payments, notifications, members, groups, users CASCADE;
```

Новые таблицы сервер создаст сам при следующем запуске.

## 3. Вписать токен в Render

Открой свой сервис на Render → **Environment** → **Add Environment Variable**:
- **Key:** `BOT_TOKEN`
- **Value:** токен из шага 1

Сохрани. (Адрес приложения `RENDER_EXTERNAL_URL` Render подставляет сам — больше ничего не нужно.)

## 4. Выкатить новую версию

Сейчас весь код Телеграм/групп — в ветке `groups`, прод (`main`) ещё на старой версии.
Сливаем и пушим:

```bash
cd ~/pay-app
git checkout main
git merge groups
git push
```

Render увидит пуш и задеплоит. В логах при старте появится строка
`Telegram: webhook → https://ТВОЁ.onrender.com/api/tg/webhook` — значит, бот подключился,
и кнопка-меню в боте настроилась автоматически.

## 5. Создать мини-аппу в BotFather (для ссылок-приглашений)

Чтобы приглашения открывались в один тап, заведём «именованную» мини-аппу:

**@BotFather** → `/newapp` → выбери **@stazhki_v_raschete_bot** →
- **Title:** Стажки в расчёте
- короткое описание и картинку — любые
- **Web App URL:** `https://ТВОЁ.onrender.com`
- **short name:** введи ровно **`app`** (это важно — фронт строит ссылки вида
  `t.me/stazhki_v_raschete_bot/app?startapp=КОД`)

## 6. Проверить на телефоне

1. Открой **@stazhki_v_raschete_bot** → внизу кнопка **«Открыть»** (меню-кнопка) → приложение
   запускается, ты сразу залогинен своим телеграм-аккаунтом.
2. Создай группу, впиши друзей по @нику (они станут призраками), занеси трату.
3. Нажми **«Пригласить»** — скопируется ссылка. Кинь её другу. Он откроет → попадёт прямо
   в эту группу и займёт своё место (если ты указал его @ник).

## Если что-то не так

- **Заглушка «Откройте через @bot» в самом Телеграме** — значит, initData не дошёл; проверь,
  что открываешь через кнопку бота, а не по обычной ссылке.
- **Приглашение не открывает группу** — проверь, что в шаге 5 short name именно `app`.
- **Бот молчит на /start** — проверь, что `BOT_TOKEN` в Render верный и деплой прошёл (в логах
  должна быть строка про webhook). Можно переразвернуть: Render → **Manual Deploy**.
- **Долгая первая загрузка** — бесплатный Render «проснулся» после простоя (~30–60 сек). Чтобы
  не засыпал, можно бесплатно настроить пинг на `https://ТВОЁ.onrender.com/api/health`
  каждые ~10 минут (например, cron-job.org).

## Локальная разработка (как раньше)

Без `BOT_TOKEN` сервер работает в dev-режиме (вход по нику/имени, без Телеграма):
```bash
cd ~/pay-app && python3 server.py    # http://localhost:8000
```
